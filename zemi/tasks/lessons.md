# Lessons Learned

ユーザーからの修正・指摘を受けた際に、再発防止のためのパターンをここに記録する。

- **「引数で渡された正しい値」と「データ構造から逆算した値」が同じはずでも食い違うことがある**: `src/engine/attackScripts.ts`の`runSshBruteforce`/`runSqlInjection`で、攻撃成功時のログ・ノード更新に関数引数`targetId`を使うべき箇所で、リンク一覧から`find`して逆算したノードIDを誤って使っていた。両者は通常は一致するが、トポロジ上のリンクの探し方(複数リンクがある場合にどれが先に見つかるか等)によっては一致しない値になり得る。**対策**: 関数の引数として明示的に渡された値がある場合は、データ構造から再導出せずそれを直接使う。並列実装(複数エージェントが同種のロジックを別々に書く場合)では特に起きやすいパターンなので、レビュー時に「同じ意味の値が2箇所以上で別経路から求められていないか」を確認する。
